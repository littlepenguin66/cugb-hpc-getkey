# v26.4.12

this release focuses on cli ergonomics, safer defaults, and better script integration.

## highlights

- fixed cli metadata so `--version` now follows the crate version automatically
- added `--password-stdin` for non-interactive and script-friendly password input
- added `--print-token` so token output is explicit instead of being tied to `--verbose`
- expanded `--status` with cache and key file details, cache state, ttl, and json output
- normalized default error messages while preserving raw detail in `--verbose` and `--json`
- help output no longer exposes the current values of `HPC_USERNAME` or `HPC_PASSWORD`

## cli improvements

- `--verbose` now focuses on step logs and diagnostic detail
- `--quiet` and `--verbose` are now mutually exclusive
- missing credentials are prompted individually instead of re-prompting both fields
- password prompt failures now return normal user-facing errors instead of panicking
- help output now includes practical examples for common flows

## scripting and status

- `ghpc --status` now reports `missing`, `invalid`, `expired`, or `valid`
- status output now includes file path, file mode, modified time, size, username, expiry time, and remaining lifetime
- `ghpc --status --json` now returns a richer machine-readable object
- success and failure json output now stay clean without mixed human-oriented logs

## upgrade note

- if you previously relied on `ghpc --verbose` to print the token, switch to `ghpc --print-token`
- for scripts, prefer `--password-stdin` over passing passwords in command arguments
- if `--status` reports `invalid`, remove `~/.hpc-login-cache.json` and run `ghpc --force` to rebuild it
