# v2026.4.11

this release focuses on reliability fixes in the rust cli and brings the bundled ai skill implementation back in sync with the main tool.

## highlights

- cached token downloads now fall back to a fresh login when the cached token no longer works
- key download failures now return real errors instead of looking like successful runs
- password encryption now reports failures cleanly instead of panicking on invalid upstream key data
- the `.skills/cugb-hpc-getkey/` implementation now matches the rust cli behavior much more closely

## cli improvements

- refactored the main login/download path into a single flow that distinguishes cache hits from fresh login
- propagate download, cache write, encryption, home directory, and file permission failures back to the caller
- save the private key only after the remote response is validated as a real success
- improve warning text when the upstream rsa public key cannot be fetched and the fallback key is used

## skill updates

- updated the bun-based skill runner to retry with fresh login when cached token download fails
- fixed skill error handling so failures exit non-zero instead of only printing to stderr
- added cache format compatibility for both camelCase and snake_case timestamp fields
- updated skill documentation to reflect the real command flow, token output behavior, install step, and security guidance

## project maintenance

- bumped the crate version from `2026.3.18` to `2026.4.11`
- expanded `.gitignore` for `.DS_Store`, `node_modules/`, and `bun.lock`

## upgrade note

- this release is a behavior-hardening update; users should see fewer silent failures when login or key download goes wrong
- if you use the ai skill package directly, run `bun install` inside `.skills/cugb-hpc-getkey/` before invoking it
